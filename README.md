# LeetCode Solutions & Revision 🚀

Welcome to my personal LeetCode solutions repository! This repository is designed to keep track of the coding problems I've solved, along with detailed explanations and time/space complexity analysis to aid in my revision.

## 📂 Repository Structure

The problems are organized by their primary data structure or algorithmic technique:

*   `Arrays/`
*   `Strings/`
*   `Linked Lists/`
*   `Trees/`
*   `Dynamic Programming/`
*   `Graphs/`

## 📝 How to add a new solution

1.  Navigate to the appropriate topic folder.
2.  Create a new folder for the problem, e.g., `0001-Two-Sum`.
3.  Copy the `Solution_Template.md` from the `Templates/` folder into this new folder and rename it to `README.md`.
4.  Add your solution code file (e.g., `solution.py`, `Solution.java`, `solution.cpp`) in the same folder.
5.  Fill out the `README.md` with the problem statement, intuition, explanation, and complexity analysis.

## 🧠 Revision Strategy

*   **Spaced Repetition:** Revisit the problems after 1 day, 3 days, 1 week, and 1 month.
*   **Focus on Intuition:** Don't memorize the code; understand the core logic and pattern used to solve the problem.
*   **Multiple Approaches:** Always try to note down the brute force approach before writing the optimized solution.
